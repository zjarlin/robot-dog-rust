# ROS2 桥接说明

这个仓库的 Rust 代码先输出结构化动作计划，后面再接 ROS2。

## 推荐方式

1. Rust 程序生成 `RobotPlan`。
2. 一个 ROS2 节点订阅这个计划。
3. ROS2 节点把计划映射到具体 topic 和 service。

## 这样做的好处

- Rust 负责安全和编排。
- ROS2 负责硬件通信。
- 两边都可以单独替换。

## 典型映射

| 动作 | ROS2 侧处理 |
| --- | --- |
| `Move` | 映射到 `/cmd_vel` |
| `Stand` | 映射到姿态控制话题 |
| `Sit` | 映射到姿态控制话题 |
| `Speak` | 触发语音播报节点 |
| `CameraScan` | 触发巡检或采图流程 |

## 备注

不同机器狗的具体话题名不一样，所以这层不要写死成某一家 SDK 的专用实现。
先保留结构化计划，再按硬件补适配器。
